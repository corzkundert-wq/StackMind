You are Replit AI Agent. Build a production-ready MVP called “Workbench AI” (internal name) that mimics the spec below AND enhances it with Deck Builder + Content Series Engine + Video Pipeline + Approval/Publish workflow. Prioritize reliability, editable outputs, and strict JSON schemas.

============================
0) STACK / PROJECT RULES
============================
- Single Replit project.
- Backend: Python + FastAPI
- Frontend: Streamlit (MVP) with clean dark theme.
- DB: Postgres + pgvector (preferred). If pgvector unavailable, fallback to Chroma (but keep interfaces identical so we can swap later).
- File storage: local filesystem in Replit (MVP) + DB metadata.
- Embeddings: OpenAI text-embedding-3-large (default) OR configurable via env var.
- LLM: OpenAI (default) OR configurable via env var.
- External integrations (optional toggles): Gamma (deck), HeyGen/Runway (video), Zapier/Make/n8n (webhooks). If no API keys, mock adapters but keep same interface.
- Environment variables: OPENAI_API_KEY, DATABASE_URL, EMBEDDING_MODEL, LLM_MODEL, APP_SECRET, WEBHOOK_DEFAULT_URL (optional), GAMMA_API_KEY (optional), HEYGEN_API_KEY (optional), RUNWAY_API_KEY (optional)
- All AI module outputs MUST be structured JSON with citations. NO freeform-only responses.

============================
1) CORE PRODUCT BEHAVIOR
============================
Workbench AI is NOT a chat-first app. It is click-module-first.
Workflow (Wizard):
A) Identity selection (required)
B) Choose Library (required)
C) Select files (required)
D) Click modules to run (Summary/Signals/Claims/Evidence/Relevance/Durability/Leverage/Canon/Decision Memo/Market Trends)
E) Vote + Pin outputs
F) Actions Panel (Deck Builder, Content Series, Email generator, Video pipeline, Export, Webhooks)
Pinned items persist within session and feed downstream outputs (Canon, Decision Memo, Deck/Series/Video).

============================
2) DATA MODEL (POSTGRES)
============================
Implement tables (with migrations or SQL init):
- libraries: id (uuid), name, description, created_at
- files: id (uuid), library_id, filename, display_name, uploaded_at, tags (jsonb), raw_text (text), status (enum: uploaded/extracted/chunked/embedded/failed), error (text)
- chunks: id (uuid), file_id, chunk_index (int), text (text), metadata (jsonb), created_at
- chunk_embeddings: chunk_id (uuid pk/fk), embedding (vector)
- identities: id (uuid), name, definition (jsonb), created_at, is_preset (bool)
- sessions: id (uuid), identity_id, library_id, created_at, selection_mode (text), selected_file_ids (jsonb)
- artifacts: id (uuid), session_id, module_name, created_at, params (jsonb), raw_output (jsonb)
- artifact_items: id (uuid), artifact_id, item_type (text), content (jsonb), confidence (float), citations (jsonb)
- votes: id (uuid), session_id, artifact_item_id, vote (enum: useful/not/partial), note (text), created_at
- pins: id (uuid), session_id, artifact_item_id, pinned_at
- approvals: id (uuid), session_id, artifact_id, status (enum: draft/reviewed/approved/scheduled/posted), scheduled_for (timestamp nullable), posted_at (timestamp nullable), channel (text nullable), meta (jsonb), updated_at

External trends tables:
- external_sources: id (uuid), tier (1/2), title, publisher, url, published_date, snippet, credibility_score, created_at
- external_clusters: id (uuid), session_id, theme, alignment_score, tier1_count, tier2_count, citations (jsonb), created_at

============================
5) MODULES (CLICK TO RUN)
============================
Each module is an endpoint and returns artifact + items. Store in DB.
- Summary: summary, key_themes, unusual_points, open_questions, recommended_followups
- Signals: insight, why_it_matters, implications, confidence, citations
- Claims: claim_text, scope_conditions, predicted_outcome, counterexample_risks, confidence, citations
- Evidence: claim_id linkage OR claim_text, supporting_citations, quote_snippet(optional), evidence_strength
- Relevance: relevance_to (identity/project/stakeholders/market), relevance_score 0-5, implication_action, confidence, citations
- Durability: horizon selection, durable_principle_candidate, durability_score 0-5, what_breaks_it, conditions, validation_test, citations
- Leverage: leverage_score 0-5, why, high_leverage_action, second_order_effects, citations
- Canon Builder: principle_statement, why_it_matters, applies_when, not_when, operator_checklist, anti_patterns, examples, citations, tags
- Decision Memo: situation_summary, key_variables, options (A/B/C), recommendation, risks_mitigations, citations
- Market Trends (Public):
  - Inputs: tier_scope (tier1/tier2/both), domain (macro/econ, industry, tech/AI, regulatory, marketing/social, geography), horizon (30/90/365)
  - Output: trend_clusters, tier1_evidence list, tier2_evidence list, alignment_score, relevance_to_identity, implications_actions
  - If no web access in Replit environment, implement “source adapter” stub + allow manual URL paste import.

Retrieval:
- For selected files: retrieve top-K relevant chunks via embeddings per module (K configurable).
- Always support “use all chunks” for small files.
